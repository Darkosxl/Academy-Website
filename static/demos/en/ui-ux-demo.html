UI vs UX

Same goal, different failure modes. Click things.

Two different jobs

UI (User Interface) is what it looks like: spacing, color, typography, affordance — does a button look clickable?

UX (User Experience) is what it feels like to use: is the flow logical, does it prevent mistakes, does it respect your time?

A screen can be gorgeous (great UI) and still infuriating to use (bad UX), or ugly (bad UI) and effortless (great UX). The next pages let you feel the difference.

UI and UX are independent axes. Click each box to simulate using it.

Good UI + Good UX Ideal

Looks great and works great.

e.g. a polished form that also validates gracefully as you type.

Good UI + Bad UX Trap

Beautiful, but the flow hides destructive actions, buries the "cancel", or resets your progress on error.

e.g. a slick modal whose only exit is a tiny mislabeled X.

Bad UI + Good UX Ugly but works

Looks like 1998, but every action is predictable, fast, and forgiving.

e.g. old craigslist — ugly, but you always know what will happen when you click.

Bad UI + Bad UX Avoid

Ugly and confusing. Everything is hard.

e.g. a form with unclear buttons that also loses your data on error.
