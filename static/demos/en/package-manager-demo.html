Every language grew its own. Same job everywhere: fetch, resolve versions, track in a lockfile.

📦 npm

JavaScript / Node.js

The default — ships with Node. Largest registry on earth (~3M packages). Everything else in JS-land is a reaction to it.

npm install express

⚡ pnpm

JavaScript / Node.js

Exists because npm copies the same files into every project. pnpm stores one global copy, hard-links it — faster installs, huge disk savings.

pnpm add express

🧶 Yarn

JavaScript / Node.js

Born at Facebook when npm was slow and non-deterministic. Invented the lockfile idea npm later adopted. Still common in older codebases.

yarn add express

🐍 pip

Python

Python's standard installer, pulls from PyPI. Usually paired with virtual environments so projects don't fight over versions.

pip install requests

🦀 Cargo

Rust

Widely considered the best-designed one: package manager, build tool, and test runner in a single coherent command.

cargo add serde

🍺 Homebrew

macOS / Linux system apps

Same idea, different target: installs whole programs onto your machine instead of libraries into a project.

brew install ffmpeg

💎 Bundler / gem

Ruby

Ruby's pair: gem installs, Bundler pins exact versions per project. Its Gemfile.lock popularized lockfiles everywhere.

bundle add rails

☕ Maven / Gradle

Java / JVM

Enterprise heavyweights — dependency management plus a full build pipeline. Gradle is the newer, faster, less-XML one.

gradle build
